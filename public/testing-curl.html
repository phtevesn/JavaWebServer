Testing with curl

curl is a command-line tool for making HTTP requests. It's perfect for testing and debugging your server.

Basic Requests

# Simple GET request curl http://localhost:8080/ # Request a static file curl http://localhost:8080/index.html # Test the echo endpoint curl http://localhost:8080/echo/1000

Viewing Headers (Verbose Mode)

# See full HTTP exchange curl -v http://localhost:8080/echo/100 Output: > GET /echo/100 HTTP/1.1 # Request > Host: localhost:8080 > User-Agent: curl/7.68.0 > < HTTP/1.1 200 OK # Response < Content-Type: text/plain < Content-Length: 100 < X-Payload-Hash: abc123... < Connection: keep-alive <

Viewing Just Response Headers

# HEAD request (headers only, no body) curl -I http://localhost:8080/echo/100 # Or with GET but showing headers curl -i http://localhost:8080/echo/100

Custom Headers

# Send Connection: close curl -H "Connection: close" http://localhost:8080/echo/100 # Multiple custom headers curl -H "Connection: keep-alive" \ -H "User-Agent: MyClient/1.0" \ http://localhost:8080/echo/100

Testing Keep-Alive

# curl reuses connections in HTTP/1.1 by default # Multiple URLs use same connection curl http://localhost:8080/echo/100 \ http://localhost:8080/echo/200 \ http://localhost:8080/echo/300 # Force new connection for each request curl --no-keepalive http://localhost:8080/echo/100

Testing Errors

# Test 404 curl http://localhost:8080/nonexistent.html # Test invalid echo size curl http://localhost:8080/echo/not-a-number # Test payload too large curl http://localhost:8080/echo/200000000

Timing Information

# Show timing breakdown curl -w "\nTime: %{time_total}s\n" \ http://localhost:8080/echo/1000000 # Detailed timing curl -w "DNS: %{time_namelookup}s\nConnect: %{time_connect}s\nTotal: %{time_total}s\n" \ -o /dev/null -s \ http://localhost:8080/echo/100

Saving Response to File

# Save to file curl -o response.txt http://localhost:8080/echo/1000 # Or redirect curl http://localhost:8080/echo/1000 > response.txt
