Welcome to the Load Test Server! This is a lightweight HTTP/1.1 server built in Java that demonstrates core networking concepts including keep-alive connections, virtual threading, and efficient request handling. It's designed for learning and performance testing.

Start with virtual threading and 30-second keep-alive timeout
java lts.java -t -k -q 8080
High-performance mode: threaded, keep-alive, and quiet

üîß Load Test Client (ltc.java)

A complete, working HTTP load testing client is provided with full source code. Use it to test your server implementation and learn how clients handle HTTP keep-alive.

What It Does:

  • Creates multiple concurrent connections using Java virtual threads
  • Sends configurable requests per connection to test keep-alive behavior
  • Verifies payload integrity using SHA-256 hash comparison
  • Reports performance metrics including latency percentiles and throughput
  • Supports connection reuse with the Connection: keep-alive header

Command-Line Usage:

java ltc.java [options]
-c <count>
Number of connections (default: 10)
-r <count>
Requests per connection (default: 1)
-s <bytes>
Payload size in bytes (default: 1000)
-d <seconds>
Delay before closing (for testing timeouts)
-h <host>
Target host (default: localhost)
-p <port>
Target port (default: 8080)

Example Usage:

java ltc.java -c 10 -r 100
10 connections, 100 requests each (1000 total requests)
java ltc.java -c 1000 -r 100 -s 10000
High-volume test: 100,000 requests with 10KB payloads
java ltc.java -c 5 -r 5 -d 2
Test server timeout: 5 connections, wait 2 seconds after requests

Why You Get The Full Source:

  • Reference implementation: See how a client implements keep-alive correctly
  • Reliable testing tool: Focus on building your server, not debugging your test client
  • Learn both sides: Understanding client behavior helps you build a better server
  • Hash verification example: See how to verify data integrity in practice

üí° Tip: Read through ltc.java to understand how it handles connection reuse, timeout detection, and hash verification. The patterns you see there will help you implement similar features in your server!
